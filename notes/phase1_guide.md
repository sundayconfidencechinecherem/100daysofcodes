# 🚀 100 Days of Code — Phase 1 (Days 1–30)

## 📌 Project-Based Learning Approach

This phase is structured around building **3 real-world projects**, each completed in **10 days**.

Instead of random practice, each project is built **section by section**, just like real-world development.

---

# 🧠 Tech Stack

- HTML5
- CSS3
- SCSS
- TailwindCSS
- JavaScript

---

# 🎯 PHASE 1 GOAL

By the end of 30 days, I will:

- Build responsive websites from scratch
- Understand layout systems (Flexbox & Grid)
- Write scalable styles using SCSS
- Use TailwindCSS efficiently
- Add interactivity using JavaScript
- Have **3 complete portfolio-ready projects**

---

# 🧱 PROJECT OVERVIEW

| Project | Days | Focus |
|--------|------|------|
| Project 1 | Day 1–10 | HTML + CSS + SCSS |
| Project 2 | Day 11–20 | TailwindCSS |
| Project 3 | Day 21–30 | JavaScript + Interactivity |

# 🚀 PROJECT 1: MODERN SAAS LANDING PAGE (SCSS)

## 🎯 Objective

Build a **fully responsive SaaS landing page** using:
- Semantic HTML
- CSS3
- SCSS (for structure & scalability)

---

## 🧩 Sections to Build

- Navbar
- Hero Section
- Features Section
- Pricing Section
- Testimonials Section
- Footer

---

## 📅 DAY-BY-DAY BREAKDOWN

### ✅ DAY 1 — Setup & HTML Structure

- Create project folder
- Create:
  - `index.html`
  - `scss/`
  - `css/`
- Write full page structure (NO styling)

---

### ✅ DAY 2 — Navbar + Hero

Build:

- Navigation bar (logo, links, CTA button)
- Hero section:
  - Headline
  - Subtext
  - Call-to-action button
  - Image/illustration

Focus:
- Flexbox
- Alignment
- Spacing

---

### ✅ DAY 3 — Features Section

Build:

- 3–6 feature cards

Focus:
- Grid layout
- Reusable styles

---

### ✅ DAY 4 — Pricing Section

Build:

- Pricing cards (Basic / Pro / Premium)

Focus:
- Card design
- Hover states

---

### ✅ DAY 5 — Testimonials Section

Build:

- Testimonial cards (image, text, name)

Focus:
- Typography
- Layout consistency

---

### ✅ DAY 6 — Footer

Build:

- Footer with links and social icons

Focus:
- Structure
- Clean spacing

---

### ✅ DAY 7 — Responsiveness

- Mobile-first adjustments
- Tablet view fixes

Focus:
- Media queries
- Layout adaptability

---

### ✅ DAY 8 — SCSS Refactor

Convert CSS to SCSS:

- Variables (colors, spacing)
- Nesting
- Partials (e.g. `_navbar.scss`, `_hero.scss`)

---

### ✅ DAY 9 — Animations & Effects

Add:

- Hover effects
- Smooth scrolling
- Subtle transitions

---

### ✅ DAY 10 — Final Polish & Deployment

- Fix UI issues
- Clean up code
- Push to GitHub
- Deploy (Netlify or Vercel)
